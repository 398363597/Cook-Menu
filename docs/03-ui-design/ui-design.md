# Cook_Menu UI 设计方案

## 1. 设计定位

Cook_Menu 第一版是个人自用菜谱本，不是社区产品。页面要服务“记录自己的做法、下次复做更稳”。

核心设计原则：

- 以食物图片为主角。
- 信息围绕复做组织，而不是围绕互动数据组织。
- 食材、调料、步骤、失败经验必须分区清晰。
- 页面保持现代极简、大留白、卡片式 UI。
- 主色使用 `#FF6B35`，辅助色使用 `#20C997`。
- 背景使用 `#FFFFFF` 和 `#F8F9FA`。
- 主文字使用 `#333333`，辅助文字使用 `#888888`。

第一版不出现任何社区化元素：关注、粉丝、获赞、厨友动态、点赞数、评论数、发布到社区、热门榜单、推荐流。

## 2. 视觉稿

可打开的 HTML 视觉稿：

```text
D:\CodeX\Cook-Menu\designs\personal-recipe-ui.html
```

视觉稿包含 4 个手机 App 画布：

- 菜谱列表页
- 菜谱详情页
- 新增 / 编辑菜谱页
- 我的 / 设置页

## 3. 底部导航

第一版底部导航只保留 3 个入口：

- 菜谱：默认首页，展示自己的菜谱列表、搜索、分类筛选。
- 新增：中间突出按钮，直接进入新增菜谱页。
- 我的：本地数据、分类管理、导入导出、设置。

不保留：

- 发现
- 社区
- 消息
- 动态
- 收藏广场

购物清单暂不进入底部导航，可作为第二版入口放在“我的 / 设置”中。

## 4. 菜谱列表页

页面目标：快速找到“我做过或想复做的菜”。

页面结构：

- 标题：`我的菜谱`
- 副标题：`记录做过的味道，下次少踩坑`
- 搜索框：搜索菜名、备注或改进点
- 分类筛选：全部、早餐、家常菜、快手菜、汤粥、面食、其他
- 菜谱卡片列表
- 新增菜谱入口

菜谱卡片展示：

- 菜谱图片
- 菜名
- 分类标签
- 用时 / 难度 / 份量
- 下次改进摘要
- 状态标签：常做 / 想尝试 / 待改进

卡片不展示：

- 点赞数
- 评论数
- 作者头像
- 发布者昵称
- 浏览量
- 收藏人数

无图片状态：

- 浅灰背景占位。
- 文案：`还没有成品图`

空列表状态：

```text
还没有菜谱
先记录一道最近做过的菜吧
[新增菜谱]
```

搜索无结果状态：

```text
没找到相关菜谱
可以换个关键词，或新建一条自己的做法
```

## 5. 菜谱详情页

页面目标：打开后能直接照着做，并看到上次失败经验。

信息顺序：

1. 菜谱图片
2. 菜名与基础信息
3. 下次改进提醒
4. 食材
5. 调料
6. 做菜步骤
7. 失败经验
8. 其他备注
9. 编辑 / 删除操作

下次改进卡片：

- 标题：`下次注意`
- 来源字段：`lessons.nextImprovements`
- 使用 `#20C997` 浅色背景。
- 空状态：`还没有记录下次改进`

食材区：

- 标题：`食材`
- 每行展示名称、用量、单位、备注。

调料区：

- 标题：`调料`
- 必须和食材分开展示，不合并为“用料”。

步骤区：

- 标题：`步骤`
- 使用纵向有序步骤列表。
- 每一步包含步骤正文、可选步骤图片、可选计时、可选备注。

失败经验区：

- 标题：`失败经验`
- 来源字段：`lessons.failureNotes`
- 放在步骤之后。
- 使用浅橙或浅灰提醒卡。
- 空状态：`还没有失败记录`

## 6. 新增 / 编辑菜谱页

页面目标：快速记录一条完整菜谱。

表单顺序：

1. 成品图
2. 基本信息
3. 食材
4. 调料
5. 步骤
6. 失败经验 / 下次改进
7. 其他备注

成品图：

- 第一版只要求支持一张主图。
- 无图时显示 `添加成品图`。

基本信息字段：

- 菜名，必填
- 分类，单选
- 简短描述
- 难度，可选
- 用时，可选
- 份量，可选
- 状态，可选：常做 / 想尝试 / 待改进

食材编辑：

- 独立区块。
- 每条食材输入：名称、用量、单位、备注。
- 支持新增和删除条目。

调料编辑：

- 独立区块。
- 每条调料输入：名称、用量、单位、备注。
- 支持新增和删除条目。

步骤编辑：

- 使用有序步骤卡片。
- 每条步骤包含正文、可选图片、可选计时分钟、可选备注。
- 第一版支持新增和删除步骤。
- 拖拽排序可作为第二版。

失败经验 / 下次改进：

- 必须是两个独立输入框。
- 失败经验占位：`例如：上次盐放早了，越煮越咸`
- 下次改进占位：`例如：最后起锅前再放盐，先尝味道`
- 两个字段可以为空，但 UI 必须展示。

删除菜谱：

- 放在编辑页底部低强调位置。
- 点击后必须二次确认。

## 7. 我的 / 设置页

页面目标：管理本地数据和基础偏好，不做个人主页。

顶部可以展示轻量统计：

```text
我的菜谱
已记录 24 道菜
最近更新：番茄鸡蛋面
```

不要展示：

- 头像大卡
- 用户昵称体系
- 关注
- 粉丝
- 获赞
- 主页访客
- 勋章

设置分组：

- 数据管理：导出本地数据、导入本地数据、清空全部数据
- 分类管理：查看预设分类，后续支持自定义分类
- 常用筛选：默认打开分类、默认排序方式
- 关于：本地存储说明、版本信息
- 第二版入口：购物清单、复做日志、多图支持、随机吃什么

## 8. 开发交接

页面路由建议：

- `/`：菜谱列表页
- `/recipes/:id`：菜谱详情页
- `/recipes/new`：新增菜谱页
- `/recipes/:id/edit`：编辑菜谱页
- `/settings`：我的 / 设置页

字段展示要求：

- `image`：列表卡片和详情首图使用。
- `name`：列表、详情、表单必填。
- `category`：列表筛选和详情标签使用。
- `description`：列表摘要和详情简介使用。
- `ingredients`：独立食材区。
- `seasonings`：独立调料区。
- `steps`：有序步骤区。
- `lessons.failureNotes`：详情页步骤后展示，编辑页独立输入。
- `lessons.nextImprovements`：详情页靠前重点展示，编辑页独立输入。
- `notes`：详情页备注区，编辑页普通多行文本。

本地存储相关 UI：

- 数据导入 / 导出放在“我的 / 设置”页。
- 不设计登录、同步、账号绑定。
- 图片第一版按单张主图处理，步骤图片为可选增强。

第一版验收重点：

- 能新增一条包含图片、食材、调料、步骤、失败经验、下次改进的菜谱。
- 能从列表按分类找到菜谱。
- 能进入详情页照着复做。
- 能编辑失败经验和下次改进。
- 页面中不出现任何社区化元素。
