```text
D:\CodeX\Cook-Menu\index.html
```

也可以在项目目录下临时启动本地服务：

```powershell
python -m http.server 8765 -b 127.0.0.1
```

```text
http://127.0.0.1:8765/
```

## 9. DEV-PWA-002 缓存更新修复（2026-06-18）

问题：

- 原 Service Worker 使用固定 `cook-menu-pwa-v1` 和 cache-first。已安装用户命中旧 `index.html` 后不会访问网络；若 Service Worker 文件本身未变化，新应用壳可能长期无法到达。
- 原离线回退会把 `index.html` 返回给任意失败的 GET 请求，图片等非导航资源也可能收到错误的 HTML 响应。

改动：

- 缓存名升级为 `cook-menu-pwa-v2`，激活时删除旧版本缓存。
- 同源 GET 请求改为 network-first：在线时返回并更新最新资源，离线时才读取缓存。后续只更新 `index.html` 或 manifest 时，已安装用户重新打开或刷新即可取得新版。
- 安装预缓存使用 `cache: "reload"`，避免新 Service Worker 建缓存时复用浏览器 HTTP 缓存中的旧应用壳。
- 仅导航请求在离线且无精确缓存时回退到 `index.html`；其他资源不再错误回退为 HTML。
- 同步修正 `docs/04-development/deploy-android-pwa.md` 的更新说明：在线内容由 network-first 更新，缓存结构或预缓存清单变化时必须升级缓存名。

开发自测：

- 确认 Service Worker 脚本语法可解析。
- 确认安装列表中的本地文件均存在。
- 确认 fetch 路径先调用网络，失败后才读取缓存；缓存写入仅接受成功响应。

给 QA Agent 的回归步骤：

1. 在线打开应用，等待 `cook-menu-pwa-v2` 建立，确认旧 `cook-menu-pwa-v1` 已删除。
2. 保持 `service-worker.js` 不变，临时修改 `index.html` 中一处可辨识文案后刷新，确认页面取得新文案且 v2 缓存同步更新；回归后还原测试文案。
3. 停止本地 HTTP 服务后刷新，确认应用壳仍可离线打开。
4. 离线请求未缓存的图片或其他非导航资源，确认不会返回 `text/html`。
5. 回归 PWA 注册、manifest、图标和核心菜谱闭环，最终结论由 QA Agent 给出。
